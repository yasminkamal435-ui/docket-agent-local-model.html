# عقاري AI — Aqary AI

منصة تعليمية لتقييم أسعار الشقق بالذكاء الاصطناعي، عربي/إنجليزي، شغّالة بالكامل في المتصفح (Client-side فقط)، وقابلة للنشر مباشرة على **GitHub Pages** بدون أي سيرفر (Backend).

## المكوّنات

| الجزء | التقنية | جاهز أم مدرَّب هنا؟ |
|---|---|---|
| توقّع السعر | Dense Neural Network (12→16→16→1), scikit-learn `MLPRegressor` | **مدرَّب فعليًا** على 200,000 عيّنة تركيبية (لا يوجد موديل جاهز عام لهذه المهمة تحديدًا) |
| Explainable AI | تحليل حساسية (Perturbation Analysis) على نفس الموديل | مبني فوق الموديل المدرَّب |
| المساعد الذكي | `onnx-community/Qwen2.5-1.5B-Instruct` عبر `transformers.js`، متخصص عبر System Prompt كـ"محلل تقييم عقاري" | **موديل جاهز** (instruction-tuned, ~1.5B params) — أذكى وأدق بكتير من الإصدارات الأصغر |
| RAG / الاسترجاع | `Xenova/all-MiniLM-L6-v2` (sentence embeddings) عبر `transformers.js` | **موديل جاهز** |
| تحليل الصور | MobileNet v2 (ImageNet) عبر TensorFlow.js + طبقة تفسير عقاري (تصنيف/ترجمة/ملخص) مبنية يدويًا فوقه | **موديل جاهز** + طبقة قواعد متخصصة فوقه |
| **تدريب حي (`js/train.js`)** | شبكة عصبية (12→16→16→1) بتتبني بأوزان عشوائية وتتدرّب فعليًا بـ mini-batch gradient descent + momentum جوّه المتصفح، على بيانات تركيبية بتتولّد لحظيًا | **تدريب حقيقي بيحصل أثناء الاستخدام**، مش نتيجة محفوظة مسبقًا — تقدر تشوف الـ Loss بينزل Epoch بعد Epoch |
| الرسومات | Chart.js | مكتبة جاهزة |

## دقّة موديل توقّع السعر (على بيانات اختبار لم يرها الموديل)

- **R² = 0.9806**
- **متوسط الخطأ المطلق (MAE) ≈ 183,315 جنيه**
- **متوسط نسبة الخطأ (MAPE) ≈ 6.81%**
- تدريب: 170,000 عينة | اختبار: 30,000 عينة

البيانات **تركيبية بالكامل** (Synthetic)، مولَّدة بصيغة رياضية بسيطة (سعر أساسي لكل متر حسب المدينة × مضاعف التشطيب × عوامل العمر/الدور/الغرف + ضوضاء عشوائية 8%) — أنظر `data/sample_dataset.csv` لعينة من 500 صف، ونفس منطق التوليد موضّح في التعليقات هنا. **النتيجة تعليمية فقط وليست تقييمًا عقاريًا حقيقيًا.**

## التشغيل محليًا

أي سيرفر ستاتيك بسيط يكفي (المتصفح يحتاج HTTP وليس `file://` عشان ES Modules تشتغل صح):

```bash
cd apartment-ai
python3 -m http.server 8000
# افتحي http://localhost:8000
```

## النشر على GitHub Pages

1. ارفع محتوى مجلد `apartment-ai/` كـ repo على GitHub.
2. من `Settings → Pages`، اختار الفرع (`main`) والمجلد (`/root`).
3. هتبقى الصفحة متاحة على `https://<username>.github.io/<repo-name>/`.

لا يوجد أي مفتاح API أو بيانات حساسة في الكود — كل الموديلات بتتحمّل من CDN عام (jsdelivr) وتشتغل في متصفح المستخدم نفسه.

## حدود المشروع (بصراحة)

- البيانات تركيبية، مش بيانات سوق حقيقية — متستخدمش الموديل ده لقرار شراء فعلي.
- MobileNet موديل تصنيف صور عام (ImageNet) ولسه مش مدرَّب خصيصًا على صور شقق؛ طبقة "المترجم/المفسّر العقاري" فوقه (`REAL_ESTATE_LEXICON` في `image-analysis.js`) بتترجم وتصنّف مخرجاته لكن متعتمدش عليها كتقييم حالة فعلي.
- Qwen2.5-1.5B-Instruct موديل أقوى بكتير من الإصدارات الأصغر وبيديك جودة ردود أعلى بوضوح، لكنه أكبر حجمًا (~1 جيجا محمّل أول مرة) وممكن ياخد وقت أطول شوية في الأجهزة الضعيفة أو النت البطيء.

## إصلاح مهم (لو الصفحة ظهرت من غير أي تنسيق أو الأزرار فاضية)

لو فتحتي الموقع ولقيتيه من غير CSS ولا نصوص جوه الأزرار: السبب الشائع إن ملفات `css/style.css` و`js/app.js` لازم تكون في نفس المكان بالظبط زي ما `index.html` بيتوقّع (فولدرات `css/` و`js/` و`data/` جنب `index.html`، مش كل الملفات مبعثرة في نفس المستوى). لو رفعتي الملفات على GitHub بالسحب والإفلات المباشر من غير الحفاظ على الفولدرات، الروابط بتفضل بتدوّر على `css/style.css` وملقاهوش، فمتضهرش أي تنسيق خالص.

كمان، `js/chatbot.js` و`js/embeddings.js` بيحمّلوا مكتبة `transformers.js` بـ **dynamic import()** جوّه الدالة نفسها وليس في أول الملف — عشان لو تحميلها فشل (نت بطيء، حظر، إلخ)، الصفحة كلها (السلايدرز، توقّع السعر، اللوحة...) تفضل شغّالة عادي، وبس ميزة الشات أو الـ RAG هي اللي بترجع لوضع بديل (Fallback).
